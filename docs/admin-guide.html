MKJM Bookstore
Admin Guide

This guide is intended for administrators of www.mkjm.ca. It covers how to manage books, users, and orders through the admin panel. Keep this document confidential.

⚠ Admin Access Only: The admin panel is restricted to accounts with the admin role. Actions taken in the admin panel — such as deleting books or users — are permanent and cannot be undone. Proceed carefully.

Admin Panel Overview

admin.php

The admin panel is the central control interface for managing the MKJM Bookstore. From here, administrators can manage the book catalogue, user accounts, and customer orders.

📚

Books

Add, edit, and remove books from the catalogue. Manage stock, pricing, and ratings.

👥

Users

View all registered users, edit their details, assign roles, or remove accounts.

📦

Orders

View and manage customer orders placed through the store.

Accessing the Admin Panel

The admin panel is only accessible to users whose account has the admin role.

  1. 1Go to www.mkjm.ca/login.php (or click Login in the navigation).
  2. 2Enter your admin Username and Password.
  3. 3Upon successful login, you will be automatically redirected to admin.php.
Note: Regular user accounts are redirected to the homepage after login. Only admin accounts are sent to the admin panel automatically. You can also navigate to admin.php directly while logged in as an admin.
Security: Never share your admin credentials. If you believe your account has been compromised, change your password immediately via the account page and notify the other administrators.

Viewing Books

The admin panel displays the full book catalogue in a table. Each row shows the book's ID, title, author, price, stock quantity, rating, and cover image filename, along with action buttons to edit or delete.

Adding a Book

manage_book.php

To add a new book to the catalogue:

  1. 1In the admin panel, click Add New Book.
  2. 2Fill in all required fields in the book form (see field reference below).
  3. 3Click Save Book to add it to the catalogue.
  4. 4The book will immediately appear in the store for customers to browse.

Book Field Reference

FieldDescriptionRequirements
TitleThe full title of the book.Required. Max 200 characters.
AuthorThe author's full name.Required. Max 150 characters.
PriceSelling price in dollars (e.g. 14.99).Required. Up to 2 decimal places.
StockNumber of copies currently available.Required. Whole number, 0 or greater.
RatingEditorial star rating for the book.Required. Integer between 0 and 5.
Cover ImageFilename of the cover image (e.g. dune.avif).Image must be uploaded to the /images/ folder separately.
Cover Images: Images are not uploaded through the form — they are referenced by filename only. Upload your cover image to the /images/ directory on the server first, then enter the filename (including extension, e.g. my_book.avif) in the Cover Image field.

Editing a Book

To update an existing book's details:

  1. 1Find the book in the admin panel's book list.
  2. 2Click the Edit button on that book's row.
  3. 3The book form will load pre-filled with the current values.
  4. 4Make your changes to any fields.
  5. 5Click Save Book to apply the changes.
Tip: Updating stock to 0 will mark the book as out of stock on the customer-facing side.

Deleting a Book

  1. 1Find the book in the book list.
  2. 2Click the Delete button on that book's row.
  3. 3Confirm the deletion when prompted.
Warning: Deleting a book is permanent. If the book exists in any customer's cart, those cart entries will also be removed automatically due to the database foreign key constraint. Consider setting stock to 0 instead of deleting if you may want to restore the book later.

Viewing Users

The Users section of the admin panel shows a table of all registered accounts. Each row displays the user's ID, username, email address, and role.

Adding a User

Administrators can create new user accounts directly from the admin panel — this is useful for creating additional admin accounts without going through the public signup page.

  1. 1In the Users section, click Add New User.
  2. 2Fill in the username, email, and initial password.
  3. 3Select the user's Role — either user or admin.
  4. 4Click Save User.
Note: Passwords are stored as secure bcrypt hashes — they are never stored in plain text. The user should change their password after first login.

Editing a User

To update an existing user's account details or promote/demote their role:

  1. 1Find the user in the users list.
  2. 2Click Edit on their row.
  3. 3Update their username, email, password, or role as needed.
  4. 4Click Save User to apply changes.

User Roles

RolePermissions
user Can browse books, add to cart, place orders, and manage their own account. Cannot access the admin panel.
admin Full access to the admin panel. Can manage all books, users, and orders. Redirected to admin panel on login.
Caution: Granting admin role to a user gives them full control over the store including the ability to delete books and other user accounts. Only grant admin access to trusted individuals.

Deleting a User

  1. 1Find the user in the users list.
  2. 2Click Delete on their row.
  3. 3Confirm the deletion when prompted.
Warning: Deleting a user is permanent and will also delete all of their cart entries due to the database cascade constraint. This action cannot be undone. Do not delete admin accounts unless you are certain there is at least one other admin account remaining.

Managing Orders

The Orders section lets you view and manage orders placed by customers through the store.

Note: Full order tracking and status management is currently being implemented. This section will be expanded in an upcoming update to include order status updates (pending, shipped, delivered), customer details per order, and order totals.

In the interim, orders submitted by customers can be reviewed and actioned manually. Contact customers via their registered email address if clarification is needed on an order.

Security Notes

Passwords

All passwords on the site are hashed using bcrypt via PHP's password_hash() function. Plain text passwords are never stored in the database. When a user changes their password, the new hash replaces the old one immediately.

Sessions

User sessions are managed server-side via PHP sessions. Logging out destroys the session. Always log out of the admin panel when stepping away from your computer.

Database

All database queries use PDO prepared statements with parameterized inputs, which protects against SQL injection attacks. Do not modify database queries without understanding this — bypassing prepared statements introduces serious security vulnerabilities.

Admin Account Best Practices

  • →Use a strong, unique password for your admin account.
  • →Do not reuse your admin password on other sites.
  • →Change the default admin password immediately if it has not been changed yet.
  • →Do not create admin accounts for users who do not need full access.
Default Credentials: The default admin account (admin / password) included in the database schema is for development only. This password must be changed before the site is made publicly accessible.
